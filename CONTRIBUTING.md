# Contributing

Thanks for helping build the reference index for agent commerce in Latin America. A few principles keep this list useful.

## Principles

- **Vendor-neutral.** Add tools that compete with the maintainers. A list that hides competitors is marketing, not a resource. Entries earn their place on merit.
- **LATAM-relevant.** The list is scoped to Latin America: Pix and account-to-account rails, regulated cross-border, fiscal, and the agent-payment protocols as they land in the region. Global tools belong here only when they matter for building in LATAM.
- **Real and current.** Link to something that exists and works. No dead links, no "coming soon", no vaporware.
- **One entry per item.** If a project fits two sections, put it in its primary one and describe both angles in the line.
- **Honest descriptions.** Say what a thing actually does, including its limits (for example: "card-first", "US-only", "sandbox", "testnet"). No adjectives doing the work of facts.

## Format

One line per entry:

```
- [Name](https://link) - one sentence on what it is and, if it matters, its rail or its limitation.
```

Keep it plain. No emoji in entries, no marketing copy, and no em-dashes. The separator between name and description is a space, a hyphen, and a space, which keeps entries lint-clean.

## How to submit

1. Fork the repo.
2. Add your entry in the right section, alphabetical within a section unless there is a clear reason not to.
3. Open a pull request describing the entry in one line and, for a new tool, a link that proves it is real (a live demo, a published package, or docs).

Maintainers may edit descriptions for accuracy and neutrality.
